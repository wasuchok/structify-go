# structify-go

`structify-go` is a small Go CLI that turns JSON into Go struct definitions with `json` tags automatically.

`structify-go` คือ Go CLI ขนาดเล็กที่ช่วยแปลง JSON ให้กลายเป็น Go struct พร้อม `json` tags แบบอัตโนมัติ

If you already have JSON and do not want to write structs field by field by hand, this tool is here to save you time.

ถ้าคุณมี JSON อยู่แล้วและไม่อยากมานั่งเขียน struct เองทีละ field เครื่องมือนี้ช่วยให้ทำงานเร็วขึ้นมาก

It can read from a file or from `stdin`, handles nested objects and arrays, and writes the generated code to `stdout` or to a file.

รองรับทั้งการอ่านจากไฟล์และ `stdin`, จัดการ object หรือ array ที่ซ้อนกันได้ และเขียนผลลัพธ์ออกทาง `stdout` หรือบันทึกลงไฟล์ก็ได้

## Features / ความสามารถ

- Converts JSON objects into ready-to-use Go structs / แปลง JSON object เป็น Go struct ที่พร้อมใช้งาน
- Reads input from `-input` or piped `stdin` / รับอินพุตจาก `-input` หรือ pipe ผ่าน `stdin`
- Writes output to the terminal or to a file with `-output` / แสดงผลทางหน้าจอหรือบันทึกลงไฟล์ผ่าน `-output`
- Preserves original JSON keys in `json` tags / เก็บชื่อ key เดิมของ JSON ไว้ใน `json` tags
- Supports primitive values, nested objects, arrays, `null`, and empty arrays / รองรับข้อมูลพื้นฐาน, object ซ้อน, array, `null` และ empty array
- Sorts fields alphabetically so output stays deterministic and diff-friendly / เรียง field ตามตัวอักษรเพื่อให้ output คงที่และเทียบ diff ง่าย
- Reuses nested struct definitions when the shapes match / ใช้ nested struct ซ้ำเมื่อโครงสร้างเหมือนกัน
- Formats generated Go code with the Go standard library / จัด format โค้ดที่สร้างด้วย standard library ของ Go

## Installation / การติดตั้ง

Build it locally in this project:

สร้างไบนารีไว้ใช้ในโปรเจกต์นี้:

```bash
go build -o structify-go ./cmd/structify-go
```

Or install it into your Go bin directory:

หรือติดตั้งลงใน Go bin directory:

```bash
go install ./cmd/structify-go
```

## Quick Start / เริ่มใช้งาน

Read from a file / อ่านจากไฟล์:

```bash
structify-go -input examples/sample.json -name UserResponse
```

Read from `stdin` / อ่านจาก `stdin`:

```bash
cat examples/sample.json | structify-go -name UserResponse
```

Write the result to a file / บันทึกผลลัพธ์ลงไฟล์:

```bash
structify-go -input examples/sample.json -name UserResponse -output model.go
```

Choose a custom package name / กำหนด package name เอง:

```bash
structify-go -input examples/sample.json -name UserResponse -package models -output model.go
```

File / ไฟล์: `examples/sample.json`

```json
{

      "tags": ["go", "json"]
```

## Example Output / ตัวอย่างผลลัพธ์

```go
// Code generated by structify-go; DO NOT EDIT.

package main

type UserResponse struct {
	Active   bool        `json:"active"`
	Age      int         `json:"age"`
	Metadata interface{} `json:"metadata"`
	Profile  Profile     `json:"profile"`
	Projects []Project   `json:"projects"`
	Roles    []string    `json:"roles"`
}

type Profile struct {
	Email string  `json:"email"`
	Score float64 `json:"score"`
}

type Project struct {
	ID   int      `json:"id"`
	Name string   `json:"name"`
	Tags []string `json:"tags"`
}
```

## Type Mapping / การแมปชนิดข้อมูล

- `string` -> `string`
- `boolean` -> `bool`
- number without a decimal point or exponent -> `int` / number ที่ไม่มีจุดทศนิยมหรือ exponent -> `int`
- number with a decimal point or exponent -> `float64` / number ที่มีจุดทศนิยม หรือ exponent -> `float64`
- object -> nested struct / object -> nested struct ซ้อน
- array -> slice of the inferred element type / array -> slice ของชนิดข้อมูลที่อนุมานได้
- empty array -> `[]interface{}`
- `null` -> `interface{}`

## CLI Help / ข้อความ Help ของ CLI

Below is the actual help text printed by the program.

ด้านล่างคือ help text จริงที่โปรแกรมแสดง:

```text
Convert JSON into Go struct definitions.

Usage:
  structify-go -input sample.json -name UserResponse
  cat sample.json | structify-go -name UserResponse
  structify-go -input sample.json -name UserResponse -output model.go

Flags:
  -input string
        Path to a JSON file. If omitted, JSON is read from stdin.
  -name string
        Required root struct name for the generated model.
  -output string
        Path to save generated Go code. If omitted, code is printed to stdout.
  -package string
        Package name for the generated Go code. (default "main")
```

## Real Usage Examples / ตัวอย่างการใช้งานจริง

Generate output to the terminal from a file.

สร้างผลลัพธ์ออกทางหน้าจอจากไฟล์:

```bash
$ ./structify-go -input examples/sample.json -name UserResponse
// Code generated by structify-go; DO NOT EDIT.

package main

type UserResponse struct {
	Active   bool        `json:"active"`
	Age      int         `json:"age"`
	Metadata interface{} `json:"metadata"`
	Profile  Profile     `json:"profile"`
	Projects []Project   `json:"projects"`
	Roles    []string    `json:"roles"`
}

type Profile struct {
	Email string  `json:"email"`
	Score float64 `json:"score"`
}

type Project struct {
	ID   int      `json:"id"`
	Name string   `json:"name"`
	Tags []string `json:"tags"`
}
```

Read from `stdin` and write to a file.

อ่านจาก `stdin` แล้วบันทึกลงไฟล์:

```bash
$ cat examples/sample.json | ./structify-go -name UserResponse -output model.go
$ cat model.go
// Code generated by structify-go; DO NOT EDIT.

package main

type UserResponse struct {
	Active   bool        `json:"active"`
	Age      int         `json:"age"`
	Metadata interface{} `json:"metadata"`
	Profile  Profile     `json:"profile"`
	Projects []Project   `json:"projects"`
	Roles    []string    `json:"roles"`
}

type Profile struct {
	Email string  `json:"email"`
	Score float64 `json:"score"`
}

type Project struct {
	ID   int      `json:"id"`
	Name string   `json:"name"`
	Tags []string `json:"tags"`
}
```

If no JSON input is provided.

ถ้ายังไม่ได้ส่ง JSON เข้าไป:

```bash
$ ./structify-go -name UserResponse
error: no input provided: pass -input <file> or pipe JSON into stdin
```

If the JSON is invalid.

ถ้า JSON ไม่ถูกต้อง:

```bash
$ echo '{"broken":' | ./structify-go -name Broken
error: invalid JSON: unexpected EOF
```

## Project Structure / โครงสร้างโปรเจกต์

- `cmd/structify-go` - Program entry point, input reading, and output writing / จุดเริ่มต้นของโปรแกรม, การอ่าน input และการเขียน output
- `internal/cli` - CLI flags and help text / จัดการ flags และ help text
- `internal/parser` - JSON decoding and validation that the input contains a single JSON document / แปลง JSON และตรวจสอบว่า input มี JSON document เดียว
- `internal/naming` - Helpers for turning names into Go-style field and type names / ช่วยตั้งชื่อ field และ type ให้เข้ากับสไตล์ Go
- `internal/types` - Recursive schema inference for JSON values / อนุมาน schema ของ JSON แบบ recursive
- `internal/generator` - Deterministic Go code generation and formatting / สร้าง Go code และจัด format ให้เรียบร้อย

## Current Limitations / ข้อจำกัดตอนนี้

- The root JSON value must be an object / ค่า root ของ JSON ต้องเป็น object
- Array elements must follow a consistent shape or primitive type / ข้อมูลใน array ต้องมีรูปแบบสอดคล้องกัน
- For arrays of objects, inference uses the first element as the reference shape and validates the rest against it / ถ้าเป็น array ของ object ระบบจะใช้ตัวแรกเป็นต้นแบบ แล้วตรวจตัวที่เหลือเทียบกับแบบเดียวกัน
- Integer inference is based on JSON syntax, not integer size limits on every platform / การอนุมาน `int` อิงตาม syntax ของ JSON ไม่ได้อิงขนาด integer ของทุกแพลตฟอร์ม
